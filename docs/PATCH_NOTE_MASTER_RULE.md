# 📜 패치노트 및 작업 이력 마스터 프로젝트 룰 (Master Project Rules)

이 문서는 프로젝트의 모든 작업 이력이 유실되지 않고 체계적으로 기록되도록 하기 위한 **절대적인 가이드라인**입니다.
모든 에이전트와참여자는 아래 규칙을 준수하여 패치노트와 작업 로그를 작성해야 합니다.

---

## 1. 📝 패치노트 작성 원칙 (Core Principles)

1. **모든 변경 사항 기록**: 코드 한 줄, 에셋 하나라도 변경되었다면 반드시 기록합니다.
2. **한국어 작성**: 모든 설명은 **한국어**로 명확하게 작성합니다. (기술 용어는 영어 병기 가능)
3. **인과 관계 명시**: 단순한 "수정함"이 아닌, "**어떤 문제**가 있어서 **어떻게** 수정했고, 결과가 **무엇**인지"를 기록합니다.

---

## 2. 🗂️ 카테고리 분류 (Categories)

작업 내용을 아래 태그로 분류하여 가독성을 높입니다.

* `[NEW]`: 새로운 기능, 페이지, 에셋 추가
* `[MOD]`: 기존 기능의 로직 변경, 디자인 수정, 밸런스 조정
* `[FIX]`: 버그 수정, 오류 해결
* `[DEL]`: 더 이상 사용하지 않는 파일 삭제, 코드 정리
* `[DOCS]`: 문서(`task.md`, 가이드 등) 업데이트
* `[TEST]`: 테스트 코드 작성, 검증 수행

---

## 3. 📄 패치노트 포맷 (Standard Format)

`CHANGELOG.md` 또는 `walkthrough.md`에 기록할 때 아래 양식을 따릅니다.

### 버전 포맷: `v[Major].[Minor].[Patch]` (예: v0.5.1)

```markdown
## [v1.0.1] 2025-01-05 (작업 타이틀)

### 🚀 주요 변경 사항 (Highlights)
*   **장비 시스템 구현**: 크리처에게 무기/방어구를 장착할 수 있는 기능 추가
*   **Wild 월드 에셋 추가**: 그리폰, 고대 거북 이미지 생성 완료

### 📋 상세 내역 (Details)

#### 1. 장비 시스템 ([NEW])
*   **기능**: 인벤토리 아이템을 크리처 슬롯(3개)에 장착 시 스탯 자동 합산 로직 구현
*   **UI**:
    *   `CreatureView.js`: 장비 슬롯 렌더링 및 클릭 이벤트 추가
    *   장비 선택 모달 팝업 구현 (장착/해제 기능 포함)
*   **데이터**: `ArtifactData.js`에 목검(Normal) ~ 성기사의 흉갑(SSR) 데이터 15종 정의

#### 2. 에셋 관리 ([MOD])
*   `images/creatures/wild/` 폴더에 생성된 배치 4 이미지 적용
*   생성 실패한 이미지(쿼터 제한) 목록 `missing_assets_report.md`에 업데이트

#### 3. 버그 수정 ([FIX])
*   `BattleManager.js`: 속성 데미지 계산 시 `undefined` 오류 수정

### ✅ 검증 결과 (Verification)
*   [x] 로컬 서버(`http://localhost:3000`) 접속 테스트
*   [x] 목검 장착 후 공격력 상승 확인 (100 -> 110)
*   [x] 장착 해제 시 스탯 원상 복구 확인
```

---

## 4. 📂 파일 추적 가이드 (File Tracking)

어떤 파일이 수정되었는지 명확히 알 수 있도록 주요 파일 경로를 함께 기록합니다.

* **Core**: `js/core/Game.js`, `js/main.js`
* **Data**: `js/data/*.js`
* **UI**: `js/ui/*.js`, `index.html`, `css/*.css`
* **Assets**: `images/`, `audio/`

---

## 5. ⚠️ 보존 규칙 (Archives)

* **`task.md`**: 작업 진행 상황을 실시간으로 반영하며, 완료된 작업은 지우지 말고 `[x]` 체크 표시로 남겨둡니다.
* **`walkthrough.md`**: 각 세션의 최종 산출물과 검증 결과를 스크린샷/로그와 함께 보존합니다.
* **과거 이력**: 이전 버전의 중요한 코드는 주석 처리하거나 백업 파일(`*.bak`)로 남겨두어 롤백에 대비합니다 (필요 시).

---

위 규칙을 준수하여 프로젝트의 투명성과 안정성을 확보합니다.
